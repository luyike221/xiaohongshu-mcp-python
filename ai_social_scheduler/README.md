# AI Social Scheduler

AI驱动的智能社交媒体运营调度系统，支持小红书等平台的内容发布、互动管理和数据分析。

## 项目简介

本项目是一个基于AI的自动化社交媒体运营系统，通过统一的调度核心自动化管理多个社交媒体平台，实现内容创作、发布、互动、数据分析的全流程智能化。

### 核心特性

- 🤖 **AI自主驱动**：AI模型自主决策和执行运营任务
- 🔄 **事件响应**：支持用户请求、平台通知、定时任务等多种事件类型
- 📊 **数据驱动**：基于数据分析优化内容策略
- 🔌 **可扩展**：模块化架构，易于接入新平台和服务
- 📱 **多平台支持**：支持小红书、抖音、快手等平台（逐步扩展）

## 技术架构

采用**分层模块化架构**，以**AI调度核心**为中心，通过**MCP协议**统一管理各个平台服务。

### 架构层次

```
用户交互层 (CLI/Web/API)
    ↓
AI调度核心层 (事件监听、AI决策、任务调度、策略管理、状态管理)
    ↓
MCP服务层 (小红书、图视频生成、数据分析等)
    ↓
平台适配层 (浏览器自动化/API调用)
    ↓
数据存储层 (SQLite/PostgreSQL)
```

## 快速开始

### 环境要求

- Python >= 3.11
- uv (Python包管理器)

### 安装

```bash
# 克隆项目
git clone <repository-url>
cd ai_social_scheduler

# 使用uv安装依赖
uv sync

# 复制配置文件
cp config/config.example.yaml config/config.yaml
# 编辑 config/config.yaml 填入实际配置
```

### 运行

```bash
# 使用uv运行
uv run python main.py

# 或激活虚拟环境后运行
source .venv/bin/activate
python main.py
```

## 核心功能

### 1. AI自主驱动
- AI模型分析运营目标和策略
- 自动生成内容创作计划
- 调用相应的MCP服务执行任务
- 监控执行结果并自动调整策略

### 2. 事件响应机制
支持多种事件类型：
- 📥 **用户请求事件**：如"帮我写一篇关于美食的小红书"
- 🔔 **平台通知事件**：新评论、新粉丝、私信等
- ⏰ **定时任务事件**：定时发布、定时数据分析等
- 📊 **数据阈值事件**：达到某个数据指标时触发相应动作

### 3. 数据分析和优化
- 内容表现数据（阅读量、点赞、评论、转发）
- 用户互动数据（粉丝增长、互动率）
- 内容趋势分析
- 热点内容识别
